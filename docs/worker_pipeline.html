Sync Loop
Polls master every MetaTimeout. Detects model version bump: latestModelId > currentModelId. Signals syncedChan (buffered=1 — signals coalesce).
Pull Loop
Wakes on syncedChan. Downloads from blobStore keyed by Unix timestamp. Atomically replaces both MatrixFactorizationItems and ClickThroughRateModel in Pipeline.
Main Loop Triggers
Two wake conditions:
  • ticker.C — periodic schedule
  • pulledChan — new model just loaded
Whichever fires first. Buffered=1 means a fast model pull won't queue multiple recommendation runs.
Consistent Hashing
Each worker handles a stable shard of users. Add a second worker → ~50% of users migrate automatically. No central partition table.

consistent.Get(userId) == me
Skip Guards
Two early exits before any computation:
  • User inactive beyond TTL
  • Digest match = cache still fresh
Saves the majority of CPU — only active users with stale caches proceed.

Model Distribution
Model ID = Unix timestamp → monotonic, no CAS needed.

Master GC's CTR models older than current CF model ID.

blobStore.Open(strconv.FormatInt(modelId, 10))
Parallelism
parallel.Detachable(len(users), p.Jobs, ...)

All per-user work runs in a bounded goroutine pool. LLM reranker detaches from the pool during the API call to avoid blocking slots.
No Cross-Source Normalization
CF scores: raw dot product (unbounded)
Item-to-item: [0, 1]
Timestamp: 10-digit int

Only FM and LLM rankers unify these into a single score space. Pass-through leaves ordering undefined across sources.
fashion-recommend Inputs
The fashion layer contributes data, not control flow:
  • user.Labels — LLM-extracted style traits (extractor.go) used as CTR features
  • item.Labels — fashion categories, attributes, embeddings
  • Config choices — which recall sources, decay values, ranker type
The Worker pipeline machinery is 100% Gorse.
FM vs LLM Tradeoff
  • FM: ~ms latency, needs training data, uses structured labels
  • LLM: 100–500ms, zero training, semantic text understanding
  • Now: fashion-recommend uses pass-through — no ranker active
Cache Design
Worker writes → Server reads. Server holds no models, does no computation on the hot path.

Cache miss → Server falls back to real-time CF. That fallback is not this pipeline — it's a direct Gorse server lookup.
